# 📘 Hack The Box: Redeemer

> 🔗 https://app.hackthebox.com/machines/Redeemer

---

## 🎯 Цель

Подключиться к Redis-серверу и извлечь флаг из базы данных.

---

## 🧠 Ключевая идея

Эта лабораторная работа демонстрирует типичную ошибку конфигурации Redis:

- сервис Redis открыт наружу;
- отсутствует аутентификация;
- любой пользователь может подключиться и читать данные.

Главная идея — **enumeration + direct data access**.

---

## 🔍 Ход решения

### Шаг 1 — Сканирование цели

```bash
nmap -Pn -sV <TARGET_IP>
```

### Почему это работает

`nmap` определяет:
- открытые порты;
- сервисы;
- версии.

Флаг `-Pn` отключает проверку доступности (важно для HTB).  
Флаг `-sV` позволяет понять, **какой именно сервис работает**.

### Что мы узнаём

- `6379/tcp open`
- сервис `redis`

### Почему это важно

Порт **6379** — стандартный порт Redis.  
Если он открыт — база данных доступна по сети.

---

### Шаг 2 — Понимание сервиса

Redis — это **in-memory база данных**.

### Почему это важно

- данные хранятся в оперативной памяти;
- доступ к Redis = доступ к данным;
- часто используется без защиты.

---

### Шаг 3 — Подключение к Redis

```bash
redis-cli -h <TARGET_IP>
```

### Почему это работает

`redis-cli` — клиент для работы с Redis.

Флаг `-h` указывает удалённый хост.

Если нет аутентификации → доступ получаем сразу.

---

### Шаг 4 — Получение информации

```bash
info
```

### Почему это работает

Команда `info` возвращает:
- версию сервера;
- статистику;
- информацию о базах данных.

### Что мы узнаём

- версия: **5.0.7**
- есть база `db0`
- количество ключей: **4**

---

### Шаг 5 — Перечисление ключей

```bash
keys *
```

### Почему это работает

`keys *` возвращает все ключи в базе.

Это аналог `ls`, но для базы данных.

### Что мы получаем

```
numb
flag
temp
stor
```

---

### Шаг 6 — Извлечение данных

```bash
get flag
```

### Почему это работает

Redis хранит данные как `key → value`.

Команда `get` возвращает значение ключа.

## 💥 В чём уязвимость

- Redis доступен извне  
- отсутствует пароль  
- данные читаются напрямую  

Это приводит к **утечке данных без эксплуатации**.

---

## 🛡️ Рекомендации

- ограничить доступ к Redis (localhost / firewall)
- включить аутентификацию
- использовать TLS
- не хранить чувствительные данные без защиты
