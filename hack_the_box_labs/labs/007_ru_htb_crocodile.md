# 📘 Hack The Box: Crocodile (ПОДРОБНЫЙ РАЗБОР)

> 🔗 https://app.hackthebox.com/machines/Crocodile

---

## 🎯 Цель
Получить доступ к системе, используя цепочку уязвимостей:
1. Найти открытые сервисы
2. Найти утечку данных
3. Использовать эти данные для входа

---

## 🧠 Общая логика (очень важно понять)

Атака строится так:

```text
Сканирование → Найти FTP → Скачать файлы → Найти логины/пароли → Найти веб → Войти → Получить флаг
```

👉 Это классический путь пентестера

---

## 🔍 Шаг 1 — Сканирование (разведка)

```bash
nmap -sC -sV <TARGET_IP>
```

### 📌 Что делает команда:
- `-sC` — запускает стандартные скрипты (ищет уязвимости)
- `-sV` — определяет версии сервисов

---

### 📥 Результат:
```
21/tcp open  ftp     vsftpd 3.0.3
80/tcp open  http    Apache
```

---

### 🧠 Как думать:

- Есть FTP → возможно файлы
- Есть веб → возможно форма входа

👉 Уже есть 2 точки атаки

---

## 🔍 Шаг 2 — Проверка FTP

```bash
ftp <TARGET_IP>
```

Вводим:
```
Name: anonymous
Password: (пусто)
```

---

### 💥 Почему это работает?

FTP настроен неправильно:
- разрешён вход без пароля

👉 Это называется:
```text
Anonymous FTP Misconfiguration
```

---

## 📥 Шаг 3 — Скачивание файлов

```bash
get allowed.userlist
get allowed.userlist.passwd
```

---

## 🔍 Шаг 4 — Анализ файлов

```bash
cat allowed.userlist
cat allowed.userlist.passwd
```

---

### 📌 Что ты видишь:

Логины:
```
aron
admin
```

Пароли:
```
Supersecretpassword1
rKXM59ESxesUFHAd
```

---

### 🧠 ВАЖНО:

👉 Это **утечка данных (information disclosure)**

И теперь главный вопрос:

```text
А вдруг эти логины и пароли подходят для веба?
```

---

## 🔍 Шаг 5 — Поиск страниц сайта

```bash
gobuster dir -u http://<TARGET_IP> -w /usr/share/wordlists/dirb/common.txt -x php,html
```

---

### 📌 Что делает gobuster:

- перебирает скрытые страницы
- ищет админки, логины и т.д.

---

### 📥 Результат:
```
/login.php
```

---

## 🔐 Шаг 6 — Вход

Открываешь:
```
http://<TARGET_IP>/login.php
```

Пробуешь:
```
admin : rKXM59ESxesUFHAd
```

---

### 💥 Почему это сработало?

👉 Потому что:
- разработчик использовал один и тот же пароль
- это называется:
```text
Credential Reuse
```
