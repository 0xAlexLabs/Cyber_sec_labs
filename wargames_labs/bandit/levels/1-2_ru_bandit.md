# Bandit Level 1 → Level 2

## 🔗 Ссылка на задание
https://overthewire.org/wargames/bandit/bandit2.html

---

## 🧠 Цель уровня
Пароль для следующего уровня хранится в файле с именем `-`, который находится в домашней директории.

Нужно прочитать этот файл и использовать найденный пароль для входа под пользователем `bandit2`.

---

## 📌 Дано
- Текущий пользователь: `bandit1`
- Пользователь следующего уровня: `bandit2`
- Сервер: `bandit.labs.overthewire.org`
- SSH-порт: `2220`
- Место хранения пароля: файл с именем `-` в `/home/bandit1`

---

## 💻 Решение

Сначала подключаемся как `bandit1`:

```bash
ssh bandit1@bandit.labs.overthewire.org -p 2220
```

После входа смотрим список файлов в домашней директории:

```bash
ls -la
```

Должен быть виден файл с именем ровно `-`:

```text
-rw-r----- 1 bandit2 bandit1 33 ... -
```

Читаем файл через явный абсолютный путь:

```bash
cat /home/bandit1/-
```

Альтернативный рабочий способ:

```bash
cat < -
```

В выводе команды будет пароль для `bandit2`.

После этого выходим из текущей SSH-сессии:

```bash
exit
```

И подключаемся как `bandit2`:

```bash
ssh bandit2@bandit.labs.overthewire.org -p 2220
```

Вводим пароль, который был выведен из файла.

---

## 📖 Объяснение

Главная особенность уровня — необычное имя файла: `-`.

В Linux и других Unix-подобных системах символ `-` часто имеет специальное значение. Многие консольные программы используют `-` как обозначение стандартного ввода (`stdin`) или стандартного вывода (`stdout`).

Например:

```bash
cat -
```

не означает «прочитать файл с именем `-`». Для `cat` это означает «читать данные из стандартного ввода». Поэтому команда начинает ждать, пока пользователь введёт текст вручную в терминале.

Именно поэтому такая команда не решает уровень:

```bash
cat -
```

Файл существует, но команда не воспринимает `-` как обычное имя файла.

---

## ✅ Почему работает `cat /home/bandit1/-`

Команда:

```bash
cat /home/bandit1/-
```

работает, потому что `/home/bandit1/-` — это абсолютный путь к файлу.

Разбор:

- `/home/bandit1/` — домашняя директория пользователя `bandit1`
- `-` — настоящее имя файла внутри этой директории

Когда `-` является частью полного пути, он уже не воспринимается как специальное обозначение stdin. Команда получает конкретный путь к файлу, поэтому `cat` спокойно читает его содержимое.

Это один из самых понятных и надёжных способов пройти уровень.

---

## ✅ Почему работает `cat < -`

Команда:

```bash
cat < -
```

работает из-за перенаправления стандартного ввода.

Символ `<` говорит shell-оболочке:

> Возьми содержимое файла после `<` и передай его команде как стандартный ввод.

То есть shell открывает файл с именем `-` и передаёт его содержимое в `cat` через stdin.

В этой команде:

- `cat` запускается без имени файла в аргументах
- `< -` говорит оболочке взять ввод из файла с именем `-`
- `cat` получает содержимое файла через stdin и печатает его на экран

Важная деталь:

Перенаправление обрабатывает shell ещё до запуска `cat`. Поэтому самой программе `cat` не нужно разбирать `-` как имя файла.

---

## ⚠️ Почему `cat ./-` обычно работает

Ещё один распространённый вариант:

```bash
cat ./-
```

Здесь `./-` означает «файл с именем `-` в текущей директории».

Разбор:

- `.` — текущая директория
- `/` — разделитель пути
- `-` — имя файла

Так как `-` становится частью пути, `cat` обычно воспринимает его как обычный файл. Во многих Linux-окружениях этот способ работает корректно.

Но в твоей сессии этот вариант не сработал, а сработали эти два:

```bash
cat /home/bandit1/-
cat < -
```

Это нормально. Главное, что оба рабочих варианта убирают неоднозначность, которая есть в команде `cat -`.

---

## ✅ Ещё один правильный способ: `cat -- -`

Часто для таких случаев используют:

```bash
cat -- -
```

Аргумент `--` означает:

> Закончить обработку опций. Всё, что идёт дальше, считать обычными аргументами.

Это полезно, когда имя файла начинается с `-`, потому что многие команды воспринимают такие значения как опции.

В этой команде:

- `cat` — команда
- `--` — конец списка опций
- `-` — имя файла, которое теперь должно быть воспринято как аргумент

Этот подход особенно полезен при работе с файлами, имена которых похожи на опции команд.

---

## 🧩 Ключевая идея

Проблема не в правах доступа. Файл доступен для чтения группе `bandit1`:

```text
-rw-r----- 1 bandit2 bandit1 33 ... -
```

Это означает:

- Владелец файла: `bandit2`
- Группа файла: `bandit1`
- Группа имеет право на чтение
- Пользователь `bandit1` может прочитать файл

Настоящая проблема — само имя файла: символ `-` имеет специальный смысл в консольных командах.

---

## 📚 Используемые команды

- `ssh` — подключение к удалённому серверу Bandit
- `ls -la` — просмотр файлов, включая скрытые, с подробной информацией о правах
- `cat` — вывод содержимого файла
- `<` — перенаправление содержимого файла в стандартный ввод
- `exit` — выход из текущей SSH-сессии

---

## 🧠 Что важно запомнить

- Файл может называться почти как угодно, даже `-`
- Простая команда `cat -` читает stdin, а не файл с именем `-`
- Можно использовать полный путь, чтобы убрать неоднозначность:

```bash
cat /home/bandit1/-
```

- Или перенаправление ввода:

```bash
cat < -
```

- Или остановить обработку опций через `--`:

```bash
cat -- -
```
